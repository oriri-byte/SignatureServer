# 要件定義書：スマートコントラクト用 ECDSA署名サーバ

## 1. 目的
ウォレットアプリからのリクエストを受け付け、指定された「アドレス」と「ドメイン名」に対して、サーバ側で厳重に管理された秘密鍵を用いてECDSA署名を生成・返却する。
本サーバは、スマートコントラクトへのドメイン登録トランザクションに必要な「正当な署名」を提供する役割を担う。

## 2. システム構成・前提条件
- **クライアント（ウォレットアプリ）**: ユーザの入力を受け付け、本サーバに署名生成をリクエストする。サーバから返却された署名を用いてトランザクションを構築し、ブロックチェーンへ送信する。
- **署名サーバ（本システム）**: データのバリデーション・署名生成を行い、クライアントへ返す。

## 3. 機能要件

### 3.1. 署名生成API
- **機能**: クライアントから「アドレス」と「ドメイン名」を受信し、ECDSAアルゴリズムによるデジタル署名を作成して返却する。
- **エンドポイント (仮)**: `POST /api/v1/sign`
- **リクエスト形式 (JSON)**:
  ```json
  {
    "address": "0x1234...",
    "domain": "example.domain"
  }
